# Platform Adapters

Adapters change field shape, not worldview. Keep the same `core_claim`, `viewer_takeaway`, and resource promise.

## YouTube

- Title: clear and searchable; can use one strong concept plus one concrete outcome.
- Description: intro first, then supplement, then chapters, then links.
- Tags: use searchable topic terms; do not stuff unrelated terms.
- Chapters: include when timestamps are available.

## Bilibili

- Title: close to YouTube but more conversational if useful.
- Description: intro first; supplement can be more detailed.
- Tags: prefer topic and tool words.
- CTA: can be softer and personal, but no generic "一键三连" unless Kun asks.

## Xiaohongshu

- Title: shorter, stronger first line. Avoid clickbait that changes the claim.
- Body: first two lines should state the problem and useful result.
- Hashtags: fewer, focused, and natural.
- Supplement: keep scannable; long resource explanations should point to the link.

## WeChat Channels

- Title/first line: concise and human.
- Body: short intro plus one supplement note.
- CTA: only if the link path actually works in the intended distribution channel.

## Douyin

- Title/caption: short and punchy, but still grounded.
- Hashtags: topic-level, not broad traffic bait.
- Supplement: usually minimal; move longer material to a comment or profile link if confirmed.

## X

- Text: one compact claim, one concrete observation, one link if available.
- Avoid long platform-style descriptions.

## Notion/Internal Record

- Preserve the complete spine, all variants, asset paths, links, and publish status.
- Notion copy can be operational and archival; it does not need platform tone.
