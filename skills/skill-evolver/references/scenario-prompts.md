# 场景提示词

## 目的

本文件用于存放“体量大且提示词密集”的场景模板，避免 `SKILL.md` 膨胀。
仅在对应场景触发时按需加载。

## 管理者总姿态

在所有场景中，先切换为“强 AI 智能体技能管理者”身份：

```text
你的职责不是给技能补几段文字，而是管理一个技能的生命周期。

先做四件事：
1) 读取当前技能的 `SKILL.md`、相关 `references/`、相关 `scripts/`，理解已有自演化资产。
2) 判断本次是新增需求，还是一次技能调用后的复盘沉淀。
3) 决定每条新收获应该进入 `SKILL.md`、`references/`、`scripts/`，还是只保留为当次证据。
4) 优先重写既有资产；只有在现有结构无法承载时，才新增文件或章节。

你的目标是让技能更强、更稳、更会自我积累，而不是让文件越来越多。
```

## 场景 1：新增能力但禁止尾部追加

当用户提出“给现有技能增加新能力”时使用。

```text
任务：对现有技能执行增量更新。

约束：
1) 禁止只在末尾追加说明。
2) 先重写受影响的既有区块。
3) 保持 `SKILL.md` 为资源路径真源。
4) 大体量场景提示词迁移到 `references/`，并在 `SKILL.md` 增加路由。
5) 输出必须包含：差分摘要、影响地图、最小补丁清单、验证证据。

输入：
- 目标技能路径：<path>
- 能力差分：<delta>

输出要求：
- 说明重写了哪些既有区块。
- 列出新增/删除/更新的资源。
- 给出验证命令与执行结果。
```

## 场景 2：调用后复盘沉淀提示词 / 注意点 / 脚本

当一次技能调用已经结束，且你发现这次对话中出现了可复用提示词、漏检点或脚本机会时使用。

```text
任务：把一次真实技能调用转化为下一轮可复用的技能资产。

执行姿态：
1) 先阅读本次调用中的用户目标、关键提示词、遗漏点、返工点、验证证据；若有真实实验，再补读根因信号、改前/改后产物、备选方案比较与当前声明边界。
2) 只提炼“未来还会复用”的内容；一次性上下文不要沉淀。
3) 把改变主行为或判断边界的内容回写到 SKILL.md。
4) 把提示词模板、提示词技巧、漏检清单、特定变体经验沉淀到 `references/`。
5) 把稳定、重复、可验证的步骤沉淀到 `scripts/`。
6) 对实证排查类对话，优先抽象“如何拿证据、如何比较替代方案、为什么选择低侵入方案、哪些话不能提前说满”。
7) 临时路径、单次截图、客户专属文件名只保留为当次证据，不写进长期资产；真正要沉淀的是采证模板与判断规则。
8) 若已有参考资料/脚本可承载，优先重写；不要机械新建文件。
9) 若本轮来自 Computer Use / 桌面取证 / 聊天阅读，对话要先按“主线消息”和“可点击分支”拆开看；凡是 `chat history`、图片、文件预览这类会改变结论的载体，必须点开看完、关闭，再回到主线继续。
10) 对这类复盘，必须把观察结果分成三类：`有效信息`、`误发或无效信息`、`工具/宿主异常与恢复策略`。误发图、无关截图、串单内容不能升格为需求；工具异常只能沉淀为操作护栏和恢复动作。

输出要求：
- 给出“观察 → 抽象 → 落位 → 预期收益”的映射。
- 若本轮包含实证排查，额外给出“根因证据 → 备选方案 → 推荐方案 → 声明边界”的映射。
- 明确列出哪些内容被忽略，以及忽略原因。
- 至少提供一组改前/改后对照，证明这次复盘沉淀不是噪音。
- 若本轮属于桌面取证或聊天阅读，额外给出“主线消息 / 可点击分支 / 有效信息 / 无效信息 / 环境异常”的五栏映射。

### 场景 2A：Computer Use / 聊天取证复盘专用补充

当一次真实调用的难点在“边滑边看”和“点开分支”之间切换时，使用这段补充提示：

```text
任务：把一次桌面聊天取证的执行经验沉淀成下一次可直接复用的阅读规则。

强制动作：
1) 先确认主线推进顺序：当前聊天 -> 向下阅读 -> 遇到可点击载体先点开 -> 看完关闭 -> 回主线继续。
2) 把 `chat history`、图片、文件预览都视为会影响结论的分支，而不是可选旁路。
3) 不把“已经看过缩略图”算作“已经完成取证”；需要点开看到正文或大图。
4) 把本轮收获严格分为：
   - 有效信息：客户真实想改的内容
   - 误发或无效信息：发错图、串单信息、与当前结论无关的材料
   - 环境异常：例如 MOS 导致滚动方向反转、窗口状态误判、点击后没有回到原锚点
5) 环境异常必须同时沉淀恢复动作，例如“改用拖动滚动条小步向下”“每次关闭分支后回到原锚点再继续”，而不是只记录出错现象。

输出要求：
- 用“主线推进规则 + 分支查看规则 + 分类贴签规则 + 环境恢复规则”四段式输出。
- 明确哪些是下次还会复用的规则，哪些只是本次对话的噪音。
```

## 场景 3：将高体量提示逻辑从 SKILL.md 拆到 references

当 `SKILL.md` 过长，或混入大量变体专属提示词时使用。

```text
任务：重构技能以改善渐进式披露。

步骤：
1) 识别 SKILL.md 中提示词密集且场景专属的区块。
2) 将这些区块迁移到 references/<topic>.md。
3) 在 SKILL.md 用简洁路由语句替换原内容。
4) 确保迁移后行为规则没有丢失。
5) 验证所有迁移后的路径都已在 SKILL.md 显式声明。
```

## 场景 4：新增确定性门控脚本

当更新质量依赖确定性检查、不能只靠启发式判断时使用。

```text
任务：为增量更新新增确定性门控检查。

门控要求：
- 检测 append-only 倾向
- 检测声明缺失资源
- 检测孤儿资源
- 检测占位符泄漏

交付物：
- scripts/<gate-script>.py
- SKILL.md 中的脚本入口命令
- 带 pass/fail 结果的验证证据
```

## 场景 5：方法论级认知回灌建议（不引用绝对路径）

当本次复盘产生了方法论级认知（换一个技能也成立的判断标准、设计原则）时使用。

```text
任务：把方法论级认知整理成回灌建议，==不在技能里写笔记绝对路径==。

执行姿态：
1) 把可复用判断流程嵌入 SKILL.md 正文（精炼后的核心 5-10 行），不引用外部笔记。
2) 把方法论级洞察整理成"话题 + 一段话总结 + 适用场景"，写进输出契约的回灌建议段。
3) 由用户判断该洞察值不值得回灌到笔记，回灌操作由用户自行执行。
4) 技能本身只承载执行所需的判断流程，不承担"知识库索引"的角色。

输出要求：
- 回灌建议清单（话题 / 一段话总结 / 适用场景）
- 不写绝对笔记路径，由用户决定笔记结构
```

## Computer Use / 聊天取证复盘的额外规则

复盘来自 Computer Use、聊天取证或桌面巡检时，演化要多做这几件事。

**采集期两项检查**：
- 分支是否走完：凡是会影响结论的可点击载体（`chat history`、图片、文件预览）是否都已点开、读完、关闭，并回到主线继续。
- 标签是否贴对：把收获分成"有效信息 / 误发或无效信息 / 工具或宿主异常"。工具异常（如 MOS 导致滚动方向反转）只能沉淀为恢复策略，不能误写成用户需求。

**资产化两类规则**：
- 阅读顺序：`主线推进 -> 点开可点击分支 -> 看完关闭 -> 回主线继续`。
- 分类框架："有效信息 / 误发或无效信息 / 环境异常与恢复策略"。

宿主或工具坑点只在"下次还会复现、且有明确恢复动作"时才进长期资产；例如"MOS 可能让滚动方向反转，优先改用拖动滚动条小步推进"可沉淀为恢复策略，不能写成用户需求。

**输出**：报告本轮识别出的"有效信息 / 误发或无效信息 / 环境异常与恢复策略"分别是什么，哪些进长期资产、哪些只留当次证据。
