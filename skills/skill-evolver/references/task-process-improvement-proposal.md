# 任务过程到技能改进建议

## 目的

把外部项目里的 `session trajectory -> skill improvement proposal` 转成中文离线维护机制：

```text
一次完整任务过程记录
  -> 技能改进建议
```

这不是运行时门禁，也不是每次任务结束都强制 review。它只用于系统维护：当一次或多次真实任务暴露出技能 / 插件的重复问题时，先生成中文可审查建议，再由用户或维护者决定是否进入正式修改。

## 输入：一次完整任务过程记录

一份过程记录至少包含：

```yaml
用户目标:
实际调用的 Skill 或 runtime exception:
关键输入材料:
读取的文件:
运行的命令:
失败或返工点:
最终结果:
验证证据:
用户后来纠正或取消的范围:
```

如果只有一句主观感受，没有过程证据，不生成改进建议。

## 五类可沉淀信号

| 信号 | 说明 | 常见建议 |
|---|---|---|
| 误触发 | 某技能被反复选中，但真实任务属于另一技能 | 修改描述字段 / 增加负边界 |
| 漏触发 | 用户多次描述同类任务，但现有技能没被调用 | 增加触发关键词或新建技能候选 |
| 重复手写 | 多次手写同一种检查表、模板、转换步骤 | 沉淀到参考资料或脚本 |
| 资源漂移 | SKILL.md 引用不存在资源，或资源存在但无人声明 | 走 skill-modify 做资源对齐 |
| 无合适技能 | 多次任务都只能靠临场组织 | 生成新技能 / 新插件候选 |

单次偶然问题、客户专属信息、一次性路径、临时调研结论，不进入长期资产。

## 输出：技能改进建议

```yaml
建议类型: 修改已有 Skill | 新增 Skill | overlay patch | upstream issue/PR | 清理资源 | 不沉淀
目标资产:
证据来源:
重复性判断: 单次偶发 | 多次重复 | 高置信系统性问题
问题分类: 误触发 | 漏触发 | 重复手写 | 资源漂移 | 无合适 skill | 其他
建议改动:
建议落位: SKILL.md | references | scripts | assets | overlay manifest | upstream | 不落位
不建议自动执行的原因:
下一步路由: skill-modify | skill-evolver | skill-creator | plugin-manager | 暂不处理
```

## 边界

- 只产出建议，不自动改 canonical 插件。
- 不把这套机制并入 `req-suite` 的任务执行流程。
- 不把这套机制扩散到业务插件的日常调用门禁。
- 如果用户明确取消某类插件吸收，建议中必须记录为 `out_of_scope`，不得绕回实施。
