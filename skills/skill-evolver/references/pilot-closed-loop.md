# 技能演化 pressure scenario 闭环

用于验证已有技能的增量差分是否真的带来行为改善。演化不是“把复盘写进去”，而是让下次同类任务更稳定地做对。

pressure scenario 必须触发本轮新增能力，并覆盖至少一个旧边界风险：误触发、漏触发、资源未读、输出契约缺失或声明过满。不要用只检查格式的 toy prompt 当验收。

## 必做条件

以下情况至少产出 1 个 pressure scenario；涉及主观质量或复杂协作时产出 2 个：

- `new-requirement` 给技能加了新能力。
- `conversation-harvest` 把一次任务经验回灌到技能。
- 本轮改变了判断顺序、输出契约、验证门、资源分流或触发边界。

如果当前会话无法验证新版运行时，输出 `pending-user-pilot` 和新对话提示词。真实新对话触发、runtime 可见性和分发验证交给 TASK-007；当前任务不能用本线程模拟结果冒充通过。

## 最小小实验简述

```text
pressure scenario 简述

待测技能：<技能 id 和路径>
变更类型：演进
差分来源：新能力需求 / 对话复盘
本轮差分要解决什么：<一句话>
旧版缺口、失败模式或复盘证据：<摘要>
新版应该新增的能力表现：<一句话>
测试提示词：<能触发新增能力的真实用户风格小任务；包含路径、约束或相邻技能干扰>
预期差异：
- <新版必须新增、避免或区分的具体点>
评判方式：
- 通过：<新增能力稳定出现且未破坏旧边界>
- 部分通过：<新增能力出现但不稳定或边界模糊>
- 失败：<新增能力未出现或引入更大误判>
当前状态：通过 / 部分通过 / 失败 / pending-user-pilot
TASK-007 输入：<若 pending-user-pilot，把本块原样交给 TASK-007>
```

## 新对话提示词

```text
你是独立技能 pressure scenario 评审者，不参与刚才的演化。请在真实新对话中判断新版技能是否自然触发，并且这次差分是否真的让技能在同类任务里更好用。

待测技能：
<技能名称和路径；overlay 技能写 registry id>

本轮差分来源：
<新能力需求或对话复盘，附一句来源说明>

旧版缺口：
<1-3 条旧版做不到或做不稳的点>

新版必须体现的新增能力：
<1-3 条必须观察到的行为差异>

请执行这个小任务：
<真实用户风格提示词；应能触发新增能力>

输出格式：
1. 评估结论：通过 / 部分通过 / 失败
2. 证据：<具体输出/行为证据>
3. 回归风险：<是否破坏旧边界>
4. 给技能维护者的反馈：<下一轮应改 SKILL.md、参考资料、脚本、description 还是触发边界>
```

## 回灌

- 通过：记录差分有效，并按需进入发布/分发验证。
- 部分通过：只修能解释不稳定的最小点。
- 失败：先判断是差分不可复用、落位错误、触发失败，还是用户需求理解错误；再决定继续演化或回退。
- `pending-user-pilot`：不声称行为闭环；把 pressure scenario 交给 TASK-007 做真实新对话测试。
